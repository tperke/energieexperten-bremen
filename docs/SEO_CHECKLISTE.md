# SEO Prüfliste

## Bereits technisch umgesetzt

1. Individueller Seitentitel und individuelle Beschreibung je Hauptseite
2. Genau eine Hauptüberschrift je normaler Seite
3. Serverseitig ausgegebene Inhalte
4. Kanonische Adressen
5. Open Graph Angaben und großes Vorschaubild
6. Strukturierte Daten ohne Bewertungen oder erfundene Unternehmensangaben
7. Brotkrumennavigation auf Unterseiten
8. Dynamische XML Sitemap
9. Robots Datei
10. Eigene Fehlerseite mit Statuscode 404
11. Lesbare Adressen über Apache Rewrite Regeln
12. Filterseiten mit `noindex`
13. Keine automatisch veröffentlichten Stadtteilseiten
14. Keine externen Ressourcen
15. Interne Links mit beschreibenden Texten

## Vor Veröffentlichung prüfen

1. Bevorzugte Domain und HTTPS Weiterleitung auf Produktivserver
2. Jede kanonische Adresse mit tatsächlicher Domain
3. Statuscodes für normale Seiten, Weiterleitungen und Fehlerseiten
4. Alle strukturierten Daten im Schema Markup Validator
5. Relevante Seiten im Google Test für Rich Results
6. XML Sitemap ohne nicht freigegebene Inhalte
7. Keine direkten PHP Adressen in internen Links oder Suchergebnissen
8. Seitentitel auf Länge und eindeutige Suchintention
9. Alternativtexte nach Austausch oder Ergänzung von Bildern
10. Google Search Console erst nach vollständiger rechtlicher Freigabe
11. Core Web Vitals mit Felddaten beobachten
12. Monatliche Prüfung zeitkritischer Förderseiten oder bei bekanntgegebenen Änderungen

## Redaktionelle Kontrolle

1. Keine unbelegten Superlative
2. Keine Garantien für Förderung, Einsparung oder Rangfolge
3. Keine künstlichen Ortsnennungen
4. Keine kopierten Anbietertexte ohne Prüfung
5. Sichtbares Aktualisierungsdatum bei Fachbeiträgen
6. Verantwortliche Redaktion und tatsächlichen Prüfer nennen
7. Quellen direkt an zeitabhängigen Inhalten auffindbar machen
